Conception du projet

Le projet IFRI MentorLink est une application web permettant la mise en relation des étudiants pour du mentorat académique et professionnel.

  • Architecture : Client-Serveur
  • Frontend : HTML, CSS, JavaScript (Bootstrap/Tailwind)
  • Backend : Python avec Django
  • Base de données : PostgreSQL
  • Modules : Gestion des comptes, Matching mentors-mentorés, Messagerie instantanée

Conception détaillée de la base de données

La base de données du projet IFRI MentorLink est relationnelle et comporte les tables suivantes :

  • Utilisateurs : id, nom, prénom, email, téléphone, mot_de_passe, filière, compétences, lacunes, disponibilités
  • Offres : id, utilisateur_id, matière, disponibilité, format
  • Demandes : id, utilisateur_id, matière, disponibilité, format
  • Messages : id, expediteur_id, destinataire_id, contenu, date_envoi

Relations principales :

  • Un utilisateur peut publier plusieurs offres et demandes.
  • Un utilisateur peut envoyer et recevoir plusieurs messages.
  • Les tables offres et demandes sont liées à utilisateurs par une clé étrangère.

Instructions de déploiement

  1. Cloner le dépôt GitHub : git clone https://github.com/nom-du-groupe/PIL1_2526_32.git
  2. Installer les dépendances : pip install -r requirements.txt
  3. Configurer la base de données PostgreSQL (importer structure.sql)
  4. Lancer le serveur Django : python manage.py runserver
  5. Accéder à l’application via http://localhost:8000

Manuel d’utilisation

Après inscription et connexion :

  • Compléter son profil (compétences, disponibilités, filière)
